

## ЧАСТЬ 1. ПРОТОКОЛ В ЭТОМ ПРОЕКТЕ (CAN-native, рабочий эталон)

> Важно: для `MotorTester_v1` и референса `working_example.txt` используется **CAN-native** формат, а не UART-кадры `FA/FB` внутри payload.

### 1.1 Формат CAN кадра (используется в коде)
- `CAN ID` = адрес мотора (обычно `0x001..0x7FF`)
- `CAN DATA` = `[cmd][data...][crc]`
- `crc = (can_id + sum(data_without_crc)) & 0xFF`

Пример (запрос статуса `F1` для ID=1):
- TX data: `F1 F2` (т.к. `0x01 + 0xF1 = 0xF2`)

### 1.2 Ответы
- Ответ приходит тем же `CAN ID`.
- Первый байт ответа — код команды (`cmd`), затем payload, затем `crc`.

## ЧАСТЬ 1B. UART-кадры FA/FB (альтернативный формат из мануалов)

### 1.1 Структура пакета данных
Связь осуществляется по принципу «Мастер (ПК/Контроллер) -> Слейв (Мотор)».

**Пакет от Мастера к Мотору (Downlink):**
| Заголовок | Адрес | Команда (Функция) | Данные | Контрольная сумма (CRC) |
| :---: | :---: | :---: | :---: | :---: |
| `FA` (1 байт) | `Addr` (1 байт) | `Code` (1 байт) | `Data` (0-N байт) | `CRC` (1 байт) |

**Ответ от Мотора к Мастеру (Uplink):**
| Заголовок | Адрес | Команда (Функция) | Данные | Контрольная сумма (CRC) |
| :---: | :---: | :---: | :---: | :---: |
| `FB` (1 байт) | `Addr` (1 байт) | `Code` (1 байт) | `Data` (0-N байт) | `CRC` (1 байт) |

### 1.2 Нюансы адресации и CRC
*   **Диапазон адресов:** `01` – `FF` (1 - 255). По умолчанию `01`.
*   **Широковещательный адрес (Broadcast):** `00`. Если отправить команду на адрес `00`, её выполнят все моторы на линии, но **ни один мотор не отправит ответ** (чтобы не забить шину).
*   **Расчет CRC (8-bit Checksum):** Это сумма всех байтов пакета (заголовка, адреса, команды и всех байт данных), обрезанная до 1 байта (логическое И с `0xFF`).
    *   *Пример:* `FA 01 80 00` -> `FA + 01 + 80 + 00 = 17B`. `17B & FF = 7B`. Итоговый пакет: `FA 01 80 00 7B`.

### 1.3 Длинный пакет данных (Long Data Package)
Служит для одновременной отправки РАЗНЫХ команд до 5 моторам в одном пакете (для синхронного старта).
*   **Формат:** `FC` + `[Блок Мотора 1]` + `[Блок 2]` + `[Блок 3]` + `[Блок 4]` + `[Блок 5]` + `CRC` (всего 52 байта).
*   Каждый блок длится 10 байт: `Адрес (1 байт) + Команда (1 байт) + Данные + нули для выравнивания`. Если адрес мотора совпадает в разных блоках, выполнится только первая команда. На длинный пакет моторы не отвечают.

## ЧАСТЬ 2. СПРАВОЧНИК КОМАНД (HEX)

*Примечание: Типы данных передаются старшим байтом вперед (Big-Endian), если не указано иное.*

### ГРУППА 1: Чтение параметров (Коды 30 - 3E, F1)

| Команда (Hex) | Описание | Формат данных ответа (Payload) | Нюансы |
| :--- | :--- | :--- | :--- |
| **`30`** | Чтение энкодера (с переносом) | `carry` (int32), `value` (uint16) | `value` от 0 до 0x3FFF (16383). При переполнении `carry` увеличивается/уменьшается на 1. |
| **`31`** | Чтение энкодера (накопительное) | `value` (int48) | Абсолютное положение вала. 1 оборот = 0x4000 (16384). |
| **`32`** | Чтение текущей скорости | `speed` (int16) | В RPM. >0 вращение CCW, <0 вращение CW. |
| **`33`** | Чтение полученных импульсов | `pulses` (int32) | Кол-во шагов, полученных от контроллера. |
| **`34`** | Чтение статуса IO портов | `status` (uint8) | Битовая маска: Bit 3 = OUT_2, Bit 2 = OUT_1, Bit 1 = IN_2, Bit 0 = IN_1. |
| **`39`** | Чтение ошибки угла вала | `error` (int32) | Ошибка = (Желаемый угол) - (Реальный угол). 51200 единиц = 360 градусов. |
| **`3A`** | Чтение статуса пина EN | `enable` (uint8) | 1 = Включен (Enabled), 0 = Выключен. |
| **`3B`** | Статус функции "Возврат в 0 при запуске"| `status` (uint8) | 0: В процессе, 1: Успех, 2: Ошибка. |
| **`3D`** | Сброс защиты по заклиниванию | `status` (uint8) | 1: Успешный сброс, 0: Ошибка сброса. |
| **`3E`** | Статус защиты по заклиниванию | `status` (uint8) | 1: Защита сработала (мотор заблокирован), 0: Всё ок. |
| **`F1`** | Запрос статуса мотора | `status` (uint8) | 0: ошибка, 1: остановлен, 2: ускоряется, 3: замедляется, 4: макс. скорость, 5: ищет дом (homing), 6: калибруется. |

### ГРУППА 2: Настройка параметров (Коды 80 - 8F, 9A, 9E, 3F)

*Эти команды изменяют настройки. При успехе мотор возвращает `01`, при ошибке `00`.*

| Команда | Аргумент (Данные запроса) | Описание и допустимые значения |
| :--- | :--- | :--- |
| **`80`** | `00` | **Калибровка энкодера.** Мотор должен быть без нагрузки! Возвращает: 0 - в процессе, 1 - успех, 2 - ошибка. |
| **`82`** | `mode` (uint8) | **Режим работы.** 0: Pulse Open, 1: Pulse Close, 2: Pulse vFOC (по умолч.), 3: Serial Open, 4: Serial Close, 5: Serial vFOC. *Для команд движения по CAN нужен режим 3, 4 или 5!* |
| **`83`** | `current` (uint16) | **Рабочий ток (мА).** 42D/28D/35D: до 3000 мА. 57D: до 5200 мА. |
| **`84`** | `micstep` (uint8) | **Дробление шага (Микрошаг).** От `00` до `FF` (от 1 до 256). |
| **`85`** | `enable` (uint8) | **Активный уровень EN.** 00: Low (Низкий), 01: High (Высокий), 02: Hold (Всегда включен). |
| **`86`** | `dir` (uint8) | **Направление вращения.** 00: CW (По часовой), 01: CCW (Против). *Влияет только на импульсный режим (Step/Dir).* |
| **`87`** | `enable` (uint8) | **Автоотключение OLED экрана.** 01: Вкл (гаснет через 15 сек), 00: Выкл. |
| **`88`** | `enable` (uint8) | **Защита от заклинивания ротора.** 01: Вкл, 00: Выкл. |
| **`89`** | `enable` (uint8) | **Интерполяция микрошага (MPlyer).** 01: Вкл (до 256 для плавности), 00: Выкл. |
| **`8A`** | `idx` (uint8) | **CAN bitrate index** (в CAN-native профиле). Типовые значения в софте: 0=125k, 1=250k, 2=500k, 3=1M. |
| **`8B`** | `addr` (uint16, 11-bit) | **Адрес мотора (CAN ID).** В runtime проекта поддерживается диапазон `0x001..0x7FF` (1..2047). `0` (broadcast) не использовать как персональный адрес. |
| **`8C`** | `respon`, `active` | **Настройка ответов.** respon (0/1) - отвечать ли на команды. active (0/1) - активно слать статус завершения движения (02/03). В `MotorTester_app` запись параметров 8C заблокирована policy-guard (защита телеметрии). |
| **`8D`** | `addr` (uint16, 11-bit) | **Групповой адрес.** В runtime проекта поддерживается диапазон `0x001..0x7FF` (1..2047). Позволяет объединить несколько моторов в группу; на групповой адрес моторы не отвечают. |
| **`8E`** | `enable` (uint8) | **MODBUS-RTU.** 01: Включить, 00: Выключить. |
| **`8F`** | `enable` (uint8) | **Блокировка кнопок экрана.** 01: Заблокировать, 00: Разблокировать. |
| **`9A`** | 5 байт (см. ниже) | **Режим 0_Mode** (Автовозврат в 0 при подаче питания). `mode` (0:Выкл, 1:DirMode, 2:NearMode), `set_zero` (0/1), `speed` (0-4), `dir` (0:CW, 1:CCW). |
| **`9B`** | `holdMa` (uint8) | **Ток удержания.** `01`-`08` (10% - 90% от рабочего тока). *Работает только в режимах OPEN и CLOSE. В vFOC ток адаптивный!* |
| **`9E`** | `enable` (uint8) | **Ремаппинг портов концевиков.** (Для Serial режима). IN_1 становится EN, IN_2 становится DIR. 01: Вкл, 00: Выкл. |
| **`3F`** | Нет данных | **Сброс к заводским настройкам.** После сброса требуется перезагрузка и калибровка энкодера (`80`). |

### ГРУППА 3: Управление и Движение (Serial Control Mode)
**Важно:** Для выполнения команд `F6`, `FD`, `FE`, `F4`, `F5` мотор должен быть переведен в один из Serial-режимов (SR_OPEN, SR_CLOSE, SR_vFOC) командой `82`.

**Параметр `acc` (Ускорение) во всех командах движения:**
Значение от `0` до `255`. 
*   `acc = 0`: Мгновенный старт/остановка (без плавности).
*   `acc > 0`: Плавное ускорение. Формула времени шага ускорения: `t2 - t1 = (256 - acc) * 50 мкс`. Чем **больше** число `acc`, тем **быстрее** мотор разгоняется.

| Команда | Описание | Формат запроса | Нюансы |
| :--- | :--- | :--- | :--- |
| **`F3`** | Включение/Отключение мотора (Enable) | `en` (uint8): 00 = Отключить (расслабить), 01 = Включить (удержание). | |
| **`F7`** | Экстренная остановка (E-Stop) | Без данных. Возвращает статус (0: фейл, 1: успех). | ⚠️ Не использовать при скорости > 1000 RPM (может повредить механику). |
| **`90`** | Настройка Homing (Поиск дома) | `HmTrig`(уровень концевика 0/1), `HmDir`(0 CW, 1 CCW), `HmSpeed`(uint16), `EndLimit`(вкл/выкл лимит). | Если `EndLimit=1`, при касании концевика мотор отключит удержание вала. |
| **`91`** | Выполнить Homing | Нет данных | Концевик механически подключается к EGND и IN_1. |
| **`92`** | Установить текущую позицию как "0" | Нет данных | Виртуальный Homing без физического движения. |
| **`F6`** | **Speed Mode (Вращение с заданной скоростью)** | 3 байта данных. Байт 1: Бит 7 = Направление (**0 = CCW, 1 (0x80) = CW**), Биты 6-4 = 0 (Rev), Биты 3-0 = Старшие биты скорости (speed[11:8]). Байт 2: Младшие биты скорости (speed[7:0]). Байт 3: Ускорение `acc`. | Скорость от 0 до 3000 RPM. Если скорость задать = 0, мотор плавно остановится. Максимальные скорости: OPEN=400, CLOSE=1500, vFOC=3000. |
| **`FF`** | Сохранить/Сбросить настройки скорости (Speed Mode) | `state` (uint8): `C8` = Сохранить, `CA` = Очистить. | При сохранении мотор будет крутиться с этой скоростью сразу после подачи питания. |
| **`FD`** | **Position Mode 1 (Относительное движение в импульсах)** | То же, что в `F6` (3 байта) + 4 байта `pulses` (uint32). | Поворачивает мотор на заданное кол-во импульсов относительно текущего положения. |
| **`FE`** | **Position Mode 2 (Абсолютное движение в импульсах)** | `speed` (uint16), `acc` (uint8), `absPulses` (int32). | Поворачивает мотор в точную координату, заданную импульсами. Значение со знаком (int32). |
| **`F4`** | **Position Mode 3 (Относительное движение по оси/углу)** | `speed` (uint16), `acc` (uint8), `relAxis` (**int24**, диапазон −8 388 607..+8 388 607). | Работает с данными абсолютного энкодера. Возможна погрешность позиционирования ±15 единиц. Рекомендуется микрошаг = 64. |
| **`F5`** | **Position Mode 4 (Абсолютное движение по оси/углу)** | `speed` (uint16), `acc` (uint8), `absAxis` (**int24**, диапазон −8 388 607..+8 388 607). | Поворот в точную координату по показаниям энкодера. |

*Примечание к командам `F6, FD, F4, F5` для остановки:* Чтобы плавно остановить мотор, выполняющий эти команды, отправьте ту же команду, но укажите `speed = 0`. Если дополнительно указать `acc = 0`, мотор остановится экстренно (рывком).

## ЧАСТЬ 3. ПРОТОКОЛ MODBUS-RTU

Если функция MB_RTU включена (через меню или команду `8E`), мотор начинает понимать стандартные запросы Modbus-RTU. 

Контрольная сумма здесь — стандартный `CRC16` (младший байт, затем старший).
Поддерживаемые функции Modbus:
*   `04H` — Read Input Registers (Чтение данных)
*   `06H` — Write Single Register (Запись одной настройки/команды)
*   `10H` — Write Multiple Registers (Используется для команд движения с множеством параметров)

### 3.1 Чтение данных (Функция `04H`)
Адреса регистров для чтения (Starting Address):
*   `0030H` (Длина 3 рег.): Чтение энкодера с переносом (Carry). Возвращает int32_t carry, uint16_t value.
*   `0031H` (Длина 3 рег.): Чтение энкодера (накопительное). Возвращает int48_t.
*   `0032H` (Длина 1 рег.): Чтение скорости (int16_t RPM).
*   `0033H` (Длина 2 рег.): Чтение полученных импульсов (uint32_t).
*   `0034H` (Длина 1 рег.): Статус IO портов (uint8_t в младшем байте).
*   `0039H` (Длина 2 рег.): Чтение ошибки угла (int32_t).
*   `003AH` (Длина 1 рег.): Статус пина EN.
*   `003BH` (Длина 1 рег.): Статус Go Home (0/1/2).
*   `003EH` (Длина 1 рег.): Статус защиты от заклинивания.
*   `00F1H` (Длина 1 рег.): Чтение статуса мотора (0-6).

### 3.2 Запись простых команд (Функция `06H`)
Адреса регистров для записи (Данные передаются как 1 байт в младшей части слова, старший `00H`, если не указано иное):
*   `003DH`: Сброс защиты по заклиниванию. (Любые данные).
*   `003FH`: Сброс к заводским. (Любые данные).
*   `0080H`: Калибровка мотора.
*   `0082H`: Установка режима работы (mode 0-5).
*   `0083H`: Установка рабочего тока (передается uint16_t в мА).
*   `0084H`: Микрошаг (micstep).
*   `0085H`: Активность пина EN.
*   `0086H`: Направление вращения.
*   `0087H`: Автовыключение экрана.
*   `0088H`: Защита от заклинивания.
*   `0089H`: Интерполяция микрошага.
*   `008AH`: Baud rate (скорость CAN).
*   `008BH`: Слейв адрес (slave address).
*   `008EH`: Вкл/Выкл MODBUS (enable 0/1).
*   `008FH`: Блокировка кнопок (enable 0/1).
*   `0092H`: Установить текущую ось в 0.
*   `009EH`: Ремаппинг концевиков.
*   `00F3H`: Enable мотора (0/1).
*   `00F7H`: Экстренная остановка.
*   `0091H`: Команда Go Home.
*   `00FFH`: Сохранить/Очистить параметры скорости (`C8H` / `CAH`).

### 3.3 Команды движения по Modbus (Функция `10H` - Запись множества регистров)
Команды движения сложнее и занимают несколько регистров.
Пример структуры для **Speed Mode (`00F6H`)**:
*   Начальный адрес: `00F6H`. Кол-во регистров: `02H` (4 байта данных).
*   Байты: `dir` (направление), `acc` (ускорение), `speed` (скорость uint16_t).

Пример структуры для **Position Mode 1 (Импульсы, относительное) (`00FDH`)**:
*   Начальный адрес: `00FDH`. Кол-во регистров: `04H` (8 байт данных).
*   Байты: `dir`, `acc`, `speed` (uint16_t), `pulses` (uint32_t).

*Для остановки движения по Modbus отправляется та же команда `10H`, но с нулевой скоростью и необходимым ускорением (`acc`).*

## ЧАСТЬ 4. ПОДВОДНЫЕ КАМНИ И ВАЖНЫЕ НЮАНСЫ

1. **Режимы работы (CR vs SR):** 
   В меню есть режимы с приставкой `CR_` (CR_OPEN, CR_CLOSE, CR_vFOC) и `SR_` (SR_OPEN, SR_CLOSE, SR_vFOC). 
   *   `CR` режимы (`0/1/2`) — не использовать для CAN-команд движения.
   *   `SR` режимы (`3/4/5`) — использовать для CAN-команд движения.
2. **Особенности vFOC режима:** 
   В режимах FOC (Field Oriented Control) рабочий ток адаптируется к нагрузке автоматически (до заданного предела). В режимах OPEN и CLOSE ток подается жестко (фиксированный).
3. **Концевики (Endstops) и Ремаппинг:**
   *   Изначально у моторов (особенно 42D) только один физический вход концевика.
   *   Функция лимитов (EndStop-limit) требует обязательной процедуры `Go Home` (`91`) после включения питания или изменения настроек. До "парковки" лимиты не работают.
   *   Если включить **Limit port remap** (`9E`), вход `IN_1` физически превращается в пин `Enable` (для внешнего контроллера), а `IN_2` в пин `Dir`. Это отключает концевики!
4. **Концевики в Serial режиме:**
   Если мотор работает по Serial (`SR_vFOC`), концевики могут аппаратно останавливать вращение (Left limit, Right limit). Для этого на пин `COM` должен быть подан соответствующий высокий уровень логики (3.3V или 5V).
5. **Адрес 00 (Broadcast):**
   Это мощный инструмент для систем с десятками моторов (например, роборука или конвейер). Вы шлете пакет на адрес `00`, и все моторы синхронно начинают движение. НО, если вы запросите позицию или статус по адресу `00`, вы получите тишину. Чтение работает только при точном указании адреса (01-FF).
6. **Выбор формата пакета:**
   Для этого проекта и драйвера используйте CAN-native: `[cmd][data...][crc]` с CRC по `can_id`.
   Формат `FA/FB` рассматривайте как альтернативный профиль из UART-описаний и не смешивайте его с текущей CAN-native реализацией.

8. **Статус мотора `F1` (нормализован):**
   `0=Fail`, `1=Stop`, `2=SpeedUp`, `3=SpeedDown`, `4=FullSpeed`, `5=Homing`, `6=Calibrating`.
7. **Абсолютные величины (int32 / int48):**
   Будьте крайне внимательны со знаками в командах позиционирования. Значения `absPulses` и `absAxis` передаются в дополнительном коде (Two's complement) для представления отрицательных значений координат. (Например, в примере мануала `-0x4000` это `FF FF C0 00`).

8. **Safety-policy по команде `8C` (защита телеметрии):**
   В production runtime `MotorTester_app` изменение `SlaveRespond/Active` через parameter patch API запрещено. Это сделано намеренно, чтобы исключить сценарий отключения ответов привода и потери телеметрии из-за ошибочной настройки.
